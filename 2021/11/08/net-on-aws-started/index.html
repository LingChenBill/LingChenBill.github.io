课程名称: Getting Started with .NET on AWS (Japanese)

1.AWS服务

如果使用无服务器计算选项，则可集中在核心应用程序上的时间变长。没有必要担心基础设施的管理和运用.

使用AWS的话，可以使用对应.NET应用程序的所有需求，广泛用途专用的存储和数据库解决方案.
强烈推荐根据应用程序所需的数据类型、访问模型、可扩展性类型选择合适的数据库.

使用身份服务的话，对于AWS云环境内的.NET应用，可以在适当的时机安全地给予适当的用户适当的访问.
使用监控和审计服务，可以获取应用程序和基础设施状态的细节，并评估定义的商业指标的性能.

2.工具

AWS的口号之一是“大楼的场所”「ビルダーのための場所」.
用户最熟悉的.NET开发工具提供扩展和整合.
这些工具是免费的，会定期更新。如果您有与AWS账户相关的认证信息，您可以使用这些工具.

  • AWS SDK for .NET : AWS服务的呼叫变得和调用对象的方法一样简单

  • AWS Toolkit for Visual Studio : 提供若干AWS服务和一些易用向导的资源管理视图

    ​ AWS Toolkit for Visual Studio是微软Visual Studio的扩展功能。这样一来，就可以使用AWS开发.NET应用程序，方便调试、部署.

  • AWS CDK for .NET : 允许云基础设施定义为代码

  • AWS Tools for PowerShell : 从命令行控制AWS服务

4.清理资源

​ 清理和删除所有资源，以避免进一步产生成本.
​ 删除Amazon RDS数据库实例.
​ 在Visual Studio的AWS资源管理器中，打开Amazon RDS节点，访问该实例.
​ 右键单击实例，选择“Delete Instance”（删除实例）。这样，Amazon RDS数据库实例将从AWS云中删除.

​

​ 删除AWS Elastic Beanstalk应用程序.
​ 在Visual Studio的AWS资源管理器中，打开AWS Elastic Beanstalk节点访问实例.
​ 右键单击实例，选择[Delete]（删除）。因此，Elastic Beanstalk的环境及其相关的所有资源将从AWS云中删除.
