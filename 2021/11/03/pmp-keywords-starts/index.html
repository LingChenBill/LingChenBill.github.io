1.监控项目工作

备选方案分析: 用于在出现偏差时选择要执行的纠正措施或纠正措施和预防措施的组合.
成本效益分析: 有助于确定最节约成本的纠正措施.
挣值分析: 对范围, 进度和成本绩效进行了综合分析.
根本原因分析: 可用于识别出现偏差的原因.
趋势分析: 根据以往结果预测未来绩效.
偏差分析: 审查目标绩效与实际绩效之间的差异.

趋势分析是一种分析技术, 它使用数学模型根据历史结果预测未来结果.
趋势分析根据以往结果预测未来绩效, 它可以预测项目的进度延迟, 提前让项目经理意识到, 按照既定趋势发展, 后期进度可能出现的问题.
趋势分析旨在审查项目绩效随时间的变化情况, 以判断绩效是正在改善还是正在恶化.

3.塔克曼阶梯理论

团队建设通常要经过五个阶段

  • 形成阶段: 团队成员刚刚加入团队, 对项目的任务和分工通常都不甚了解, 需要项目经理直截了当, 清晰明确地指明工作内容和要求.
    在这个阶段, 处于了解项目情况中, 完成的工作量是最小的.项目经理需要指导团队, 并将他们从这个阶段推进到执行阶段.
    团队成员独立工作.

  • 震荡阶段: 团队成员之间爆发冲突, 土气低落, 项目应该通过斡旋, 调解, 利用自己的影响力引导团队走出冲突的旋涡.

    团队开始了解项目工作.

  • 规范阶段: 团队成员已经开始互相认可, 彼此适应, 项目应该适度接后撤, 需要参与时再参与, 主要起到帮助和促进团队建设的作用.
    团队开始相互学习.

  • 表现阶段/成熟阶段: 团队已经配合默契, 项目经理应该充分信任团队, 给团队成员更充分的授权, 为团队提供必要的支持.
    在成熟阶段, 他们作为一个组织良好的单位工作, 达到最大的性能.

团队不同发展阶段项目经理的领导风格:

  • 形成阶段 —- 指令型/指导式
  • 震荡阶段 —- 影响型/教练式
  • 规范阶段 —- 参与型
  • 表现阶段/成熟阶段 —- 授权型/委任式

7.WBS

WBS : 100%原则, 滚动式规划, 唯一编码

滚动式规划 : 在项目生命周期的不同阶段, 工作的详细程度会有所不同. 在早期的战略规划阶段, 信息尚不够明确, 工作包只能分解到已知的详细水平;
而后, 随着了解到更多的信息, 近期即将实施的工作包就可以分解到具体的活动.
keywords : 无法继续分解, 信息不够明确, 后期规划

项目范围根据项目范围基准进行测量, 该基准由项目范围声明, 其关联的 WBS 和 WBS 词典组成.

范围基准是创建 WBS 流程的输出.

创建WBS的输入包括 :

  • 项目范围说明书

  • 事业环境因素
  • 组织过程资产

54.计算题

EV(挣值) : 对已完成工作的测量, 用该工作的批准预算来表示. 某时间点(通常为数据日期)所有已完成工作的计划价值(挣值), 与实际成本无关.

PV (计划价值) : 为计划工作分配的经批准的预算. 某时间点(通常为数据日期或项目完成日期)计划完成的工作的价值.

AC(实际成本) : 在给定的时间段内, 因执行项目活动而实际发生的成本. 某时间点(通常为数据日期)所有已完成工作的实际成本.

为了应付现金流突发情况, 在必要考虑做储备. 做储备的前提是要知道挣值与实际成本之间的差距, 才好做储备规划.
CV就是指出了实际绩效与成本支出之间的关系.

SV(进度偏差) SV = EV - PV : 在某个给定时间点, 项目与计划交付日期相比的亏空或盈余量, 表示为挣值与计划价值之差.

按计划完成 : TCPI = (BAC - EV) / (BAC - AC)

  • 上报 : 威胁不在项目范围内, 超出了项目经理的权限

  • 规避 : 项目团队采取行动来消除威胁, 或保护项目免受威胁的影响.它可能适用于发生概率较高, 且具有严重负面影响的高优先级威胁.
    风险规避涉及变更项目管理计划以消除风险. 虽然情况极端, 但关闭项目是一种激进但合法的回避策略.

  • 转移 : 对威胁的责任转移给第三方, 让第三方管理风险并承担威胁发生的影响.

    • 购买保险; 使用履约保函; 使用担保书; 使用保证书; 签订协议; 绩效债券; 担保; 固定价格合同

  • 减轻 : 采取措施来降低威胁发生的概率和影响

    • 采用较简单的流程, 进行更多次的测试, 或者选用更可靠的卖方(供应商); 原型开发;
      选择更可靠的的卖方 -> 属于减轻

  • 接受 : 承认威胁的存在, 但不主动采取措施, 用于低优先级威胁

    • 建立应急储备(主动); 定期对威胁进行审查(被动)

      项目计划不修改, 但建立应急措施, 是**主动接受**
      项目计划不修改, 且不建立应急措施, 是**被动接受**. 项目计划修改, 是规避
